# Commercial Upgrade Audit

Use this reference for a commercial-upgrade audit after a feed, source index, and asset bible exist.

## Checks

- identity drift: make sure recurring people are not split between anonymous roles, aliases, and later named identities.
- standard-name drift: verify each production name follows the evidence-backed standard name, not a convenient alias or invented label.
- period/state drift: require source evidence for age, cultivation state, injury, outfit, face, mood, or faction-state changes; keep face continuity across variants unless the source changes it.
- source-span coverage: compare feed lines to the requested source span and flag omitted setup, reactions, reveals, transitions, and hooks.
- local continuity: check each line against the previous visible state for position, prop possession, speaker, scene, and unresolved actions.
- physicalization safety: allow only motion supported by the source or direct consequence; do not escalate posture, travel, combat, prop handling, or exits.
- prompt contamination: remove copied commercial boilerplate, instruction-protection games, fixed refusal text, tool side effects, silent source replacement, fixed-duration packaging, and rewrite formulas.

## Audit Notes

- Treat identity split, alias handling, standard-name evidence, and period/state evidence as blocking issues when they change reusable assets.
- Treat source-span omission and local continuity breaks as coverage issues even when the feed still reads smoothly.
- Treat physicalization safety as a source-fidelity issue, not just visual polish.
- Treat prompt contamination as copied workflow residue: do not preserve boilerplate that changes the source, tool behavior, delivery package, duration, or refusal style.
