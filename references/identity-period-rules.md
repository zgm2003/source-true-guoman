# Identity And Period Rules

Use these rules with `agents/source-indexer.md` when source identity, alias drift, role upgrades, or outfit/state variants affect production assets.

## Identity

- Canonical name is an evidence-backed source identity. Choose the stable source name, title, or role label that has the strongest evidence in the requested scope, then cite the basis.
- Aliases are evidence, not production names. Record aliases, title drift, nickname use, OCR/ASR variants, and generic labels as evidence that supports or questions the canonical identity.
- Role tier: classify roles by production value and recurrence, speaking, or plot function. Do not promote one-off background people to reusable assets without evidence.
- first-person labels: keep first-person labels and self-references as alias evidence. Do not use them as production names unless the source confirms them as a stable name or title.
- Generic-role upgrade: when an early disciple/NPC/guard/maid/passersby/black-robed figure later gains a name, repeated function, dialogue, relationship, faction, or plot effect, bind those mentions to one reusable identity if evidence supports it.
- If two mentions may be the same role but evidence is incomplete, keep them as suspected same asset. Do not merge faces, rename assets, or claim confirmation in production output.

## Period And State

- Period/state trigger: create a period or state variant only when the source changes time period, outfit, disguise, injury, transformation, faction role, battle state, or other visible production identity.
- Each variant needs start evidence and end/change evidence when the requested scope contains it.
- Period/state choices stay tied to the read slice. A smoke-test slice may record local variants, but it must not turn them into global identity decisions.

## Scope

- Smoke-test limitation: smoke-test identities, aliases, and period/state variants are local notes until the full requested scope is read or the user explicitly accepts the limited slice.
- Formal multi-chapter indexing still requires full requested-scope pre-scan before final identity, recurrence, scene reuse, or asset-value decisions.
